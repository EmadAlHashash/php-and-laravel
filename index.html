الدليل التفصيلي لاحتراف PHP

مرجع تفصيلي يشرح الأكواد حرفاً بحرف، مصمم للمهندسين ومطورين الواجهات الخلفية (Backend Developers) للتحضير لمقابلات العمل الاحترافية.

الأساسيات

بنية الكود (Syntax)

لغة PHP تبدأ دائماً بوسم (Tag) محدد لكي يعرف الخادم أن هذا كود برمجي وليس مجرد نصوص HTML عادية.

<?php
    echo "Hello World";
?>
  • <?php
    علامة الفتح. تخبر الخادم (Server) أن ما بعدها هو كود PHP يجب معالجته وتفسيره.
  • echo
    أمر (Construct) وظيفته الطباعة أو تقديم إخراج (Output) للبيانات لكي يقفز للواجهة.
  • "Text"
    علامتي التنصيص تحيطان بالنصوص (Strings).
  • ;
    الفاصلة المنقوطة (Semicolon). تخبر المترجم بانتهاء هذا السطر. بدونها يتوقف التطبيق (Fatal Error).
الأساسيات

المتغيرات والبيانات

المتغيرات هي صناديق في الذاكرة نحفظ فيها البيانات مؤقتاً. في PHP لا تحتاج لتعريف نوع المتغير مسبقاً (Dynamically Typed).

<?php
    $age = 28;
    $name = "Ahmad";
    echo $name . " is " . $age;
  • $
    علامة الدولار إجبارية. ترشد المترجم أن الكلمة التي تليها هي متغير (Variable).
  • age
    اسم المتغير. يُمنع أن يبدأ برقم أو يحتوي مسافات.
  • =
    معامل التعيين (Assignment). نأخذ القيمة التي على اليمين ونضعها بداخل المتغير الذي على اليسار.
  • .
    معامل الدمج (Concatenation). يقوم بالصاق النصوص والمتغيرات ببعضها في مخرج واحد.

سؤال مقابلة

س: ما الفرق بين == و === في لغة PHP؟
ج: المعامل == يقارن "القيمة" فقط (يسمى Loose Comparison)، فتقييم "5" == 5 يعطي إجابة صحيحة رغم اختلاف النوع. أما === فيقارن القيمة والنوع البياناتي معاً (Strict Comparison)، وبالتالي "5" === 5 هي معادلة خاطئة (False).
الأساسيات

المصفوفات (Arrays)

المصفوفة هي متغير واحد قادر على تخزين أكثر من قيمة. في PHP نستخدم كثيراً المصفوفات الترابطية (Associative) التي تمتلك "مفاتيح" بدلاً من الأرقام.

<?php
    $user = [
        "name" => "Ahmad",
        "role" => "Admin"
    ];
    echo $user["name"];
  • [ ]
    الأقواس المربعة تُستخدم لإنشاء مصفوفة (الطريقة الحديثة بدلاً من array()).
  • "name"
    المفتاح (Key). نستخدم الأسماء بدلاً من الأرقام للوصول للبيانات بسهولة.
  • =>
    السهم المزدوج. وظيفته الوحيدة هي ربط "المفتاح" بالـ "القيمة" داخل المصفوفة.
  • $user["name"]
    طريقة استخراج البيانات؛ نقول للمترجم: اذهب للمصفوفة $user وأحضر القيمة المربوطة بالمفتاح "name".
الأساسيات

الشروط والحلقات (Control)

من أهم الحلقات التكرارية في PHP هي foreach المخصصة للمصفوفات، بالإضافة للشروط المنطقية if.

<?php
    foreach ($user as $key => $value) {
        if ($key === "role") {
            echo "Role: " . $value;
        }
    }
  • foreach
    حلقة تكرارية صُممت خصيصاً للمرور على المصفوفات عنصراً عنصراً دون الحاجة لمعرفة عددها.
  • as
    كلمة محجوزة بمعنى (على هيئة). تخبر المترجم بتفكيك المصفوفة وإعطاء كل عنصر شكل $key => $value.
  • if
    تعبير شرطي. ينفذ الكود الذي بداخله فقط إذا كان التقييم بين الأقواس true.

سؤال مقابلة

س: ما الفرق بين حلقة for و foreach ولماذا نفضل الأخيرة؟
ج: نفضل foreach مع المصفوفات لأننا لا نحتاج لمعرفة طول المصفوفة مسبقاً، وتستطيع استخراج المفاتيح (Keys) النصية بسهولة، بينما حلقة for تتعامل مع الأرقام التتابعية فقط (Indexes).
الأساسيات

الدوال (Functions)

الدوال هي أجزاء من الكود قابلة لإعادة الاستخدام. في PHP الحديثة يتم التركيز بشدة على تحديد أنواع البيانات المُدخلة والمُخرجة (Type Hinting).

<?php
    function add(int $a, int $b): int {
        return $a + $b;
    }
    echo add(5, 10);
  • function
    تعريف بداية الدالة ليتم استدعاؤها لاحقاً.
  • int $a
    (Type Hinting): نجبر المستخدم على تمرير رقم صحيح int للمتغير $a ولن نقبل نصوصاً!
  • : int
    (Return Type): إقرار صريح بأن هذه الدالة يجب أن تٌرجع (Return) رقماً صحيحاً في نهايتها حصراً.
  • return
    إنهاء الدالة وإرجاع النتيجة. بدونها الدالة ستنفذ العملية ولكنها لن تخرج بشيء يمكن طباعته.
الأساسيات

المتغيرات الشاملة (Superglobals)

هي متغيرات مبنية مسبقاً في لغة PHP تكون متاحة للوصول من أي مكان في الكود دون الحاجة لتمريرها. (مثل `$_GET`, `$_POST`, `$_SESSION`).

<?php
    session_start();
    $_SESSION["user_id"] = 55;
    echo $_GET["id"];
  • session_start()
    كلمة السر للبدء بمتغيرات الجلسة، يجب أن تكون في بداية الملف.
  • $_SESSION
    مصفوفة شاملة تحفظ البيانات في جانب الخادم (Server) لكي يتذكر المستخدم عند تنقله بين الصفحات (كالـ Login).
  • $_GET
    مصفوفة شاملة تستقبل البيانات الممررة عبر الرابط (URL). مثال لو الرابط ?id=10 ستطبع رقم 10.
الأساسيات

إدارة الأخطاء (Exceptions)

المهندس الحقيقي لا يترك تطبيقه ينهار فجأة أمام المستخدم، بل يصطاد الخطأ ويعالجه عبر `try` و `catch`.

<?php
    try {
        if ($divisor == 0) {
            throw new Exception("Division by zero!");
        }
    } catch (Exception $e) {
        echo $e->getMessage();
    }
  • try
    كتلة التجربة: اكتب هنا الكود الذي تشك أنه قد يسبب خطأ كارثياً.
  • throw new Exception
    "إطلاق" الإنذار يدوياً. يتم رمي مجسم من نوع (استثناء) مع رسالة مخصصة.
  • catch (Exception $e)
    شبكة الاصطياد: إن طار إنذار من try، سيقع فوراً هنا داخل المتغير $e لتطبع رسالته بسلام دون انهيار الموقع.
الكود المتقدم

البرمجة الكائنية (OOP)

الـ OOP هي تحويل الكود إلى كائنات (Objects) تتفاعل مع بعضها. الكائن يملك خصائص (Properties) ويقوم بأفعال (Methods).

<?php
class Car {
    public $color;

    public function __construct($c) {
        $this->color = $c;
    }

    public function drive() {
        return "Driving " . $this->color;
    }
}

$myCar = new Car("Red");
echo $myCar->drive();
  • class Car
    نقوم بتعريف القالب أو المخطط للكائن.
  • public
    صلاحية الوصول؛ تعني أن هذه الخاصية أو الدالة يمكن لأي كود من خارج وداخل الكلاس استخدامها وتعديلها.
  • __construct
    الدالة البناءة (Magic Method). تُنفذ كودها تلقائياً وفوراً بمجرد إنشاء الكائن دون الحاجة لمناداتها.
  • $this->
    تعني (هذا الكائن). وتستخدم حصراً للوصول لخصائص الكائن من داخل الكلاس نفسه. السهم -> مخصص للـ Objects فقط.
  • new
    الكلمة السحرية التي تحول القالب الجامد (Class) إلى نسخة حقيقية وملموسة بالذاكرة (Object).
الكود المتقدم

الواجهات (Interfaces)

الواجهة هي "عقد صارم". تجبر الكلاسات التي توافق عليها بأن تقوم بكتابة دوال محددة، ولكن الواجهة نفسها لا تحتوي كود تنفيذي بداخلها.

<?php
interface Logger {
    public function log($message);
}

class FileLogger implements Logger {
    public function log($message) {
        echo "Saved: " . $message;
    }
}
  • interface
    إنشاء الواجهة فقط بتعريف "اسم وتوقيع" الدوال، بدون كتابة منطق برمجي بداخلها.
  • implements
    كلمة التعهد. الكلاس FileLogger يعلن أنه سيلتزم بعقد Logger. لو لم يتم كتابة دالة log() سينهار التطبيق.

سؤال مقابلة Senior

س: ما هو الفرق الجوهري بين Interface و Abstract Class؟
ج:
Interface: لا يمكنه احتواء منطق منفذ نهائياً. والكلاس يمكنه الالتزام (implement) بعدة Interfaces معاً.
Abstract Class: يمكنه احتواء دوال منفذة بالكامل ودوال غير منفذة (Abstract). ولكن الكلاس يتاح له أن يرث (extends) من Abstract واحد فقط كحد أقصى.
الكود المتقدم

السمات (Traits)

لأن لغة PHP لا تدعم وراثة أكثر من كلاس (No Multiple Inheritance)، تم ابتكار الـ Traits كي تنسخ وتلصق مجموعة دوال بداخل أي كلاس بسرعة.

<?php
trait HasImage {
    public function uploadImage() {
        return "Uploaded";
    }
}

class Product {
    use HasImage;
}
  • trait
    الكلمة المحجوزة لتعريف حزمة الدوال. لا يمكن عمل new من السمة، فهي ليست كلاس.
  • use
    تُكتب داخل الكلاس لـ (استيراد/حقن) الدوال الموجودة بالسمة كأنها كُتبت بداخل الكلاس حرفياً.
الكود المتقدم

الدوال السحرية (Magic Methods)

الدوال التي تبدأ بشرطتين سفليتين __ وتعمل من تلقاء نفسها عند حدوث حدث معين للمجسم (Object).

<?php
class User {
    public function __toString() {
        return "This is a User Object";
    }
}
$u = new User();
echo $u;
  • __toString
    دالة سحرية تُستدعى تلقائياً إذا حاول المبرمج طباعة المجسم Object مباشرة باستخدام echo.
  • echo $u;
    في العادة طباعة المجسم مباشرة تعطي خطأ (Fatal Error)، لكن بفضل الدالة السحرية أعلاه سيتم طباعة النص المرجع.

مساحات الأسماء (Namespaces)

الـ Namespaces تشبه المجلدات العادية في جهازك. تمنع تضارب أسماء الكلاسات لو أنشأت صفين (Classes) بنفس الاسم في ملفين مختلفين.

<?php
namespace App\Models;

use App\Controllers\UserController;

class User {
    // ...
}
  • namespace
    تحدد عنوان أو "مجلد وهمي" لهذا الكلاس لتميزه عن غيره. (مثال: هذا مجسم مستخدم خاص بالمودلز وليس لوحة التحكم).
  • App\Models
    اسم العنوان. المسار الوهمي يتم كتابته مفصولاً بالشرطة المائلة الخلفية \.
  • use
    لو أردت استدعاء كلاس من عنوان (Namespace) آخر لداخل كلاسك الحالي، تستخدم كلمة الاستيراد هذه.

الخصائص والدوال الثابتة (Static)

أحياناً لا نحتاج لعمل new من الكلاس لاستدعاء دواله، بل نكلم الكلاس مباشرة. هنا يأتي دور الـ Static.

<?php
class MathHelper {
    public static $pi = 3.14;
    
    public static function square($num) {
        return $num * $num;
    }
}
echo MathHelper::$pi;
echo MathHelper::square(4);
  • static
    كلمة تُضاف للدالة أو المتغير لجعله "ثابتاً"، أي أنه يعيش بداخل قالب الـ (Class) مباشرة ولا يحتاج إلى نسخة من الكائن (Object).
  • MathHelper::
    استدعاء الكلاس مباشرة. علامة :: تُستخدم مع المتغيرات والدوال من النوع Static فقط (تغنينا عن استخدام ->).

ميزات PHP 8 و 8.1

كل مهندس يجب أن يعرف المزايا الجديدة للغات البرمجة، في PHP 8 توجد إضافات سحرية قللت من حجم الأكواد وضعفت الأداء.

<?php
// Nullsafe Operator
$country = $user?->profile?->country;

// Match Expression
$status = match($code) {
    200 => "OK",
    404 => "Not Found",
    default => "Unknown Error"
};
  • ?->
    (Nullsafe): إن كان الـ $user يمتلك ملفاً تعريفياً سيجلب البلد، أما إن كان المتغير المعطى null فسيتوقف بأمان ولن ينهار النظام.
  • match
    النسخة الصارمة والمتطورة من switch. تقوم بإرجاع قيمة مباشرة وتقارن البيانات بصرامة (===).
  • default
    القيمة الافتراضية إذا لم يتطابق الرقم المار داخل الـ Match مع أي شيء آخر.
أساسيات Laravel

بنية MVC والتوجيه

الـ Laravel يجعلك تنظم تطبيقك باستخدام نمط MVC (الموديل، الفيو، والكنترولر) مع نظام توجيه ذكي للطلبات.

// routes/web.php
Route::get('/users', [UserController::class, 'index']);
  • Route::get
    استدعاء كلاس الرابط من نوع طلب (GET). استخدام :: يسمى بالعربية (محلِّل النطاق) لطلب דوال ثابتة (Static Methods) دون الحاجة لعمل new Route.
  • '/users'
    الرابط (URI) الفعلي للمتصفح. مثال: website.com/users
  • UserController::class
    الكنترولر (C). هو العقل الذي يستقبل الطلب، يكلم الـ Model لإحضار بيانات المستخدمين، ويمررها للـ View (واجهة الـ HTML).
  • 'index'
    اسم الدالة بداخل الكنترولر التابعة لهذا الراوتر والتي ستعمل خصيصاً في هذه اللحظة لعرض الصفحة.
